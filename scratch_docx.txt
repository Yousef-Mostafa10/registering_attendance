University Attendance System
Flutter / Dart — Frontend Work Division
3 Developers  •  34 Endpoints  •  5 Controllers  •  API v1
Project Overview
This document divides the University Attendance System API (34 endpoints, 5 controllers) across 3 Flutter developers. The split is designed to balance workload fairly while grouping related screens and state management concerns together. Each developer owns a complete vertical slice of the app.
Base URL:
http://supergm-001-site1.ntempurl.com
Auth: All protected endpoints require Authorization: Bearer <JWT> header.
⚖️  Load Balancing Note
The original split gave Dev 3 a heavier load (13 endpoints). In this revised split, the Attendance reporting endpoints (lecture-report, section-report, absence-warnings, session-attendees history) have been moved to Developer 2, leaving Developer 3 with the pure real-time work (session lifecycle + live attendance submit). This makes all three loads comparable in effort.

Flutter Notes — Auth:
On successful login: save token + refreshToken + role + userName via token_storage.dart, then use go_router to redirect based on role.
DeviceId: call DeviceIdService.getDeviceId() from shared util — same value must flow into login, activate, and refresh calls.
Silent refresh: in auth_interceptor.dart, on 401 → POST /refresh-token → retry original request. If refresh returns 400/401 → clear storage → navigate to /login.
Handle 429 on both activate and login — show message and disable the submit button for 30 seconds.
TokenDto for refresh includes deviceId — server invalidates refresh token if deviceId mismatches (security kill switch).

Flutter — QR Rotation Timer:
Timer? _qrTimer;
void startQrRotation(int sessionId) {
  _qrTimer = Timer.periodic(Duration(seconds: 10), (_) async {
    final res = await sessionService.rotateQr(sessionId);
    setState(() { qrContent = res.newQr; pinCode = res.newPin; });
  });
}
void stopQrRotation() => _qrTimer?.cancel();
Flutter — SSE Live Dashboard:
// Use the http_sse package or dart:io HttpClient for SSE
final client = http.Client();
final request = http.Request('GET', Uri.parse(url));
request.headers['Authorization'] = 'Bearer $token';
request.headers['Accept'] = 'text/event-stream';
final response = await client.send(request);
response.stream.transform(utf8.decoder).listen((data) {
  final event = jsonDecode(data);
  if (event['status'] == 'closed') {
    stopQrRotation();  // Kill switch — stop timer too
    client.close();    // Close the SSE stream
  } else {
    setState(() { count = event['count']; attendees = event['attendees']; });
  }
});

Flutter Notes — Session & Attendance:
GPS: call Geolocator.requestPermission() and Geolocator.getCurrentPosition() before opening QR scanner screen.
QR scan: use mobile_scanner package. On scan, populate scannedQrContent field and auto-submit (no tap needed).
PIN entry: provide a 4-digit TextField alternative on the same screen — set sessionPIN, leave scannedQrContent null.
403 on submit = device mismatch (cheating attempt) — show a prominent AlertDialog warning, do NOT allow retry.
400 on submit = out of range, expired QR, or already recorded — show specific message based on response body text.
Stop session: call stop endpoint → cancel _qrTimer → SSE stream will receive status:closed → close the stream.
delete attendance: no need to manually refresh the live dashboard — the SSE stream auto-updates when a record is deleted.
Always dispose the SSE client and cancel the timer in the widget's dispose() method to prevent memory leaks.
